# PRD Quality Checklist

- Requirement analysis, metrics definition, scenario catalog, use cases, end-to-end flow, and artifact plan were used as inputs.
- PRD level is explicit and justified.
- Scope and non-scope are clear.
- Target users, roles, and client surfaces are clear.
- Functional requirements are testable.
- Success metrics and guardrails are linked or explicitly not applicable.
- Requirements map to scenarios and use cases.
- Flow-sensitive requirements reference `end-to-end-flow.md` or standalone diagrams.
- Exception flows are covered.
- Fields, data sources, permissions, and write-back behavior are in a field dictionary when needed.
- Acceptance criteria can be converted into test scenarios.
- Diagrams are standalone files and linked from the PRD when needed.
- Open questions are not treated as facts.
- Humanizer did not change facts, scope, or acceptance criteria.
