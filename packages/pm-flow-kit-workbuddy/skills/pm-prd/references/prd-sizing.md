# PRD Sizing

Choose the smallest PRD that can safely support downstream design, engineering, testing, and review.

## L1 Light PRD

Use for small changes with low uncertainty.

Typical cases:

- Copy or label changes.
- Small field additions.
- Single-page rule clarification.
- Local workflow adjustment without state or permission changes.

Required:

- Problem and goal.
- Scope and non-scope.
- Acceptance criteria.
- Risks or non-applicability notes.

## L2 Standard PRD

Use for normal product iterations.

Typical cases:

- Multiple scenarios or roles.
- Multiple pages or client surfaces.
- Conditional branches.
- Field, permission, or integration impact.

Required:

- Scenario summary.
- User flow summary.
- Functional requirements.
- Field and data notes.
- Exception flows.
- Acceptance criteria.
- Linked diagrams if needed.

## L3 Deep PRD

Use for high-risk or broad work.

Typical cases:

- New platform or core flow.
- Cross-system work.
- State machines.
- Money, inventory, fulfillment, approval, compliance, or AI launch.
- Major data-source or write-back behavior.

Required:

- End-to-end flow.
- State and data model guidance.
- Field dictionary.
- Permissions and audit concerns.
- Exception and rollback handling.
- Linked diagrams.
- Testable acceptance criteria.
