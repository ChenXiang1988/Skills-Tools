# Requirement Analysis Preflight

Requirement analysis is mandatory before PRD, prototype, diagram, review, or polishing output.

## Required Artifacts

- `requirement_analysis.md`
- `scenario-catalog.md`
- `use-cases.md`
- `end-to-end-flow.md`
- `metrics-definition.md`
- `artifact-plan.md`

## Required Coverage

- Original request.
- Problem and goal.
- Whether the original request is a solution proposal, plus the restated underlying problem when needed.
- Scope and non-scope.
- Confirmed facts, assumptions, and open questions.
- Requirement analysis level.
- Assumptions and risks.
- Success criteria and acceptance lens.
- Metrics definition or non-applicability reason.
- Scenario catalog at required grain.
- Use cases at required grain.
- End-to-end flow at required grain.
- Artifact routing.
- Deferred or non-applicable artifacts, premature technical stack choices, export, review, and delivery decisions.
- Source error check.

Use `pm-requirement-analysis` to produce these artifacts.
