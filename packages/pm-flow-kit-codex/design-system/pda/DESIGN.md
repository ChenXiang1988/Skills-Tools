# PDA And Mobile Design Rules

Use this extension for handheld, mobile, or task-focused prototypes.

## Rules

- Optimize for one task per screen.
- Use larger tap targets than desktop prototypes.
- Keep primary actions fixed or easy to reach.
- Show loading, empty, error, permission, and retry states when relevant.
- Avoid dense desktop tables on PDA screens; use task rows or cards instead.

## Tokens

Load `assets/design-system/tokens.css` first. When a prototype explicitly needs
PDA-specific values, also load `assets/design-system/pda/tokens.css`.
