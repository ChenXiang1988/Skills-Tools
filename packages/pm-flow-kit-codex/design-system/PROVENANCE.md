# design-system Provenance

This directory is a generic pm-flow-kit bundled asset. It does not contain
customer-specific, company-specific, or domain-specific design language.

## Scope

- Use for offline HTML prototypes generated by `pm-html-prototype`.
- Prefer a project-provided `design-system/` when one exists.
- Do not treat this directory as product knowledge or business context.
- Do not use this asset to override explicit project design guidelines.

## Override Contract

A project design system is eligible when it contains at least:

```text
design-system/tokens.css
```

Optional files such as `DESIGN.md`, `components.html`, and `pda/tokens.css`
are copied when present.

## Maintenance

Keep this asset small and generic. If a project has its own brand, component
library, or domain-specific visual language, put that in the project workspace
or knowledge base instead of editing this default.
